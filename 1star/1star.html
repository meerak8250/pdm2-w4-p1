Instructions

This sketch folder includes the same JavaScript files (and classes) that were used to create the complete grid-based collision detection example shown in lecture yesterday. Only the contents of sketch.js are different.

The player moves differently in this sketch than in the version shown in lecture. If you click on the sketch, the player will move right until it disappears. Click again and the player will reappear in its starting position.

Your task is to modify sketch.js to make the player move as shown in the video below. Here are the steps:

  1. Open sketch.js. Near the top of the file, there is a global boolean variable called isPlayerMoving. The value of isPlayerMoving is changed in mouseClicked(). Look at the draw() function and you will see that when isPlayerMoving is true, the player moves right. To complete this task, you will need to change the line player.moveRight() to something else...
  2. Create a new function with no parameters and no return. This function will eventually be used to figure out which direction the player should move in then move it, so give it a descriptive name of your choosing.
  3. In draw(), replace player.moveRight() with a call to the function you created in the previous step.
  4. Implement the logic in your new function to create the movement shown below. The logic is straightforward:
    • If the player is not on a surface, move down.
    • If the player is on a surface, move right.
    You should only need to use the Grid isOccupied() method to implement the logic.

Expected output:

If you're not sure how to use the isOccupied() method, look back at the gridBasedMazeComplete example in the example code repo. You can find it in the Lectures folder > W4L1. The code to review is in sketch.js.

You shouldn't need to make any additional changes to any of the classes in any of the other files.

You will probably notice there is sometimes a little gap between the bottom of the player and the wall it is resting on. You do not need to fix it in this practical (it's hard to do!). The gap happens because the player moves at a speed of 3 pixels at a time so, if it is 2 pixels above the surface, moving down would make it overlap with the wall.
