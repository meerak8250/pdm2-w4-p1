Instructions

This exercise assumes you are familiar with class inheritance, which was the subject of Week 2 Lecture 2 and Week 2 Practical 2. If you missed this material, you may need to go back and complete it before continuing with this exercise.

In this exercise, you will create a new type of grid that will store references to objects found in each cell, instead of the true / false values in the existing grid. This will allow the stationary objects stored in the grid to respond when the player collides with them.

  1. In stationaryObjects.js, add a new subclass of StationaryObject that will represent treasure the player can collect. The constructor of the new child class should take x, y, width, and height as arguments and should have its own colour e.g. yellow. Give each treasure a point value. Unlike Walls, which are solid (can't be passed through), the new child class should not be solid, meaning it can be passed through.
  2. In sketch.js, create a simple scene involving walls and treasure (at least one of each). For best collision detection results, place each object in a grid cell--the x and y coordinates should be multiples of 50. Feel free to use an existing scene as a starting point. For example, the sketch in the video below used the Week 4 Lecture 1 maze example as starter code.
  3. grid.js contains the Grid class discussed in lecture and used in the last exercise. If you're at all unsure of how it works, take some time to read through it.
  4. Currently, the Grid stores booleans in the #cells array. You are going to modify this so each cell stores null if the cell is empty or a child of StationaryObject if it is occupied. null is a JavaScript keyword meaning "nothing".
    1. Modify the #createEmptyGrid() method so that each cell is initially set to null instead of false
    2. Modify the addToGrid() method so that each cell that contains the obstacle argument is set to obstacle instead of true
    3. Modify the isOccupied() method so that instead of returning whatever is contained in a particular grid cell, it returns false if the cell contains null, and true if the cell contains a StationaryObject.
  5. In sketch.js, create a global variable called grid, which should be an instance of the Grid class with a cell size of 50. If you're reusing code from lecture or the previous exercise, this variable should already exist and be set up properly.
  6. Add all instances of walls and treasure to the grid by calling the addToGrid() method.
  7. Make the player move using the keyboard. The player should not be able to move through stationary objects that are solid, but it should be able to move through stationary objects that are not solid. You will probably need to add a new method to the Grid class that will return the contents of a particular cell. Feel free to make modifications to the Player class as well.
  8. When the player passes over a treasure, the player should earn points and the treasure should disappear from the canvas. You will need to modify the Player and Treasure classes.
  9. Optional extension: Add different types of stationary objects that the player can collect.

Step 1

When you create your new class, don't forget to add extends StationaryObject after the class name. This means your class will inherit all attributes and methods from the StationaryObject class.

Step 4

The line inside #createEmptyGrid() that sets the initial value of each cell looks like this:

this.#cells[row][col] = false;

Change false to null.

The line inside addToGrid() that indicates that a cell contains a stationary object looks like this:

this.#cells[row][col] = true;

Replace true with obstacle. This will store a reference to the obstacle in the 2D array that represents the grid.

The documentation for isOccupied() states that the method should return a boolean—true if there is an object in that grid cell or false if it is empty. Currently, isOccupied() returns true or false by simply returning the content of the grid cell. This made sense when the grid stored boolean values. Now that the grid stores either null or an object of type StationaryObject, you will need to write a conditional to determine whether to return true or false. Here is some pseudocode:

if the grid cell contains null:
  return false
else:
  return true

Translate this pseudocode into JavaScript.

Step 7

You will need to check if a grid cell is occupied or not and, if it is occupied, whether the object it contains is solid. The StationaryObject class contains a method called isSolid() that you can use to check if an object is solid or not.
