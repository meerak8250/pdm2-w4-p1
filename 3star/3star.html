3 Star Exercise

Your sketch:

Instructions

So far, you've been moving the player around a grid of stationary objects. In platformer games, it is often the grid that moves horiontally while the player stays in the middle of the screen. The moving background creates the illusion that the player is moving through an environment bigger than the screen.

If you refresh the page, you will see that the player object drops from the top of the canvas and stops when it hits the wall at the bottom. You can make the player jump by pressing "j" but it cannot be moved in any other way. Your task is to make the player appear to move horizontally by moving the grid, as described below.

Familiarise yourself with the code

  1. Read through the StationaryGridObject class in stationaryGridObjects.js. It is quite different from the StationaryObject class you have seen before. The main difference is that the location and dimensions are provided in terms of the grid (rows and columns) rather than pixels. You will also notice that each StationaryGridObject contains a reference to the grid it belongs to, and it is added to that grid as soon as it is created.
  2. Take time to read through the MovingGrid class in movingGrid.js. You will see that this implementation is similar to the previous Grid implementation, but it has some new static variables to represent direction and a state variable, #direction. There are also some differences in the addToGrid() and isOccupied() methods.
  3. Take a look at the PlatformPlayer class in platformPlayer.js. The PlatformPlayer can be in three possible states: jump (moving up), fall (moving down), or stop.

Make the MovingGrid moveable

In the MovingGrid class:

  1. Add two new private attrbutes, #offsetX and #speedX. #offsetX represents the x coordinate of the grid (top left corner) and should be initialised with 0. There is already a getter method for this variable. Update this method so it returns the value of #offsetX instead of a hard coded number. #speedX sets the amount the grid should move by. Set it to 3.
  2. Add a new public method called move(). This method should use #speedX to update the value of #offsetX depending on the grid's state (the #direction attribute).
  3. You will need to make use of the #offsetX variable in the isOccupied() method to make sure that the method still works correctly when the grid has been moved. This will be easier to do after you have completed the steps below. You may find it helpful to look at how the grid offset is used in the StationaryGridObject class.

In sketch.js:

  1. Go to the draw() function. There are two TODOs for you to implement. First, if the user is pressing "a", the you should set the direction of the grid so that when it moves, the player appears to move left. If they are pressing "d", set the direction of the grid so that the player appears to move right.
  2. Finally, call the grid's move() method after the grid's direction has been updated.

Test your sketch carefully to make sure that it works as shown below. The key thing is that when the user presses "a" or "d", it is the grid that moves, not the player, but it creates the effect that the player is moving.

Optional extensions

Here are some suggestions:

  1. The sketch only has one platform at the bottom. Make it more interesting by adding more platforms of different widths. New platforms can be added in the createWalls() function toward the bottom of sketch.js. The maximum jump height of the player is set in the PlatformPlayer class. Feel free to change it!
  2. Most platformers have much wider grids. Modify the MovingGrid class so it is not restricted to the size of the canvas. Then, create a wider scene of platforms.
  3. Add collectable items that the player can jump to get. The most OOP way to do this would be to create subclasses of StationaryGridObject that are collectable, like the treasure in the 2 star exercise.

Expected output:

You should not need to edit the PlatformPlayer or StationaryGridObject at all. All changes should be in sketch.js and movingGrid.js.
